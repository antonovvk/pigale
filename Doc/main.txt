 
/*! \mainpage  <b> Pigale</b>
  Pigale is an algorithm library and a graph editor which has been designed for graph theoretical research. 
  It is based over a new graph data structure optimizing topological operations on static graphs.<br> 
  It consists of three parts:
 <ul>         
  <li>Pigale core library <b>tgraph</b> includes graph algorithms and fundamental data structures. </li>
  <li>Pigale Qt graph editor <b>pigale</b> is the graphic interface of the library
   embedded in a graph editor. </li>
  <li><b>client</b> a client  which can interact with <b>pigale</b> over the network . </li>
 </ul>

   \section package Package
     This package includes the source files of the library <b>tgraph</b>, 
     the editor <b>pigale</b>, the <b>client</b> program 
      and the sources of the <b>freeglut </b>library.<br>
     The editor and the client use  the <b>Qt4</b> toolkit,   
     the <b>OpenGL</b> libraries for the 3-dimensional drawings, 
     and the library freeglut to draw text on 3<sup><small>d</small></sup> objects.<br>
 
   \subsection directories Organization of files
    In the main directory one find the following directories:
    <ul>
      <li> <b>tgraph</b>   (source code for the standalone algorithmic library) </li>
      <li> <b>freeglut</b>  (source code for the freeglut library) </li>
      <li> <b>qt</b>  (source code for the editor) </li>
      <li> <b>ClientServer</b>  (source code for the client) </li>
      <li> <b>UsingTgraph</b>  a small example on how to use the library </li>
      <li> <b>cgi</b> an example for a web server  </li>
      <li> <b>incl/TAXI</b>  (headers used by the standalone algorithmic library) </li>
      <li> <b>incl/GL</b>  (headers used by the freeglut library) </li>
      <li> <b>incl/QT</b>  (headers used by the editor) </li>
    </ul>
   <b>qt</b> contains also all the headers that need to be proceeded by <b>moc</b>.<br>
   The header  <b>incl/QT/Action.h</b> is generated by <b>awk</b> from <b>incl/QT/Action_def.h</b>
   
   \section system Platform dependency
   Pigale has been tested on <b>Linux</b>, <b>Linux</b> and <b>Windows</b>.<br>
   There is very little code platform dependant.<br>
   
   \section archi Architecture of the Editor
   Pigale uses two threads:
   <ul>
     <li> The main thread is responsible of all the graphics. </li>
     <li> The PigaleThread  is responsible of all the algorithms 
     (without any graphic action). </li>
   </ul>
   We give below a rough idea how Pigale works.<br>
   <p>
   When a menu is clicked a QAction action signal is emitted. 
   That action contains an integer which identifies the algorithm to be performed.<br>
   The signal is received by pigaleWindow::handler(QAction *qaction) who passes the identifier
   to pigaleThread.<br> 
   pigaleThread call a handler (defined in handler.h) 
   which execute the needed action and returns a value which indicates
   if some graphical needs to be performed.<br>
   The pigaleThread sends a signal to the main thread 
   telling  which graphic action should be done.<br>
   The pigaleThread sleeps until the main thread has finished its graphic action.<br>
   </p>
 */


/*! \page page1 Distances used in Pigale
  The following distances are used to embed a graph in  R<sup><small>n</small></sup>
  <br><br>  
 <table>
 <tr> 
 <td>
    \f$ \text{Czekanovski-Dice distance}  \f$ 
</td><td>                      
    \f$  \qquad d^2(i,j)=1-\frac{|N(i)\cap N(j)|}{|N(i)|+|N(j)|} \f$ 
 </td></tr>  

 <tr>
 <td> 
  \f$ \text{Oriented distance} \f$  
  </td><td> 
   \f$  \qquad d^2(i,j)=1-\frac{|N^-(i)\cap
    N^-(j)|}{|N^-(i)|+|N^-(j)|}-\frac{|N^+(i)\cap
    N^+(j)|}{|N^+(i)|+|N^+(j)|}     \f$ 
 </td> </tr>

 <tr>
 <td> 
 \f$ \text{Adjacency distance (not Euclidean)} \f$ 
</td><td>            
  \f$ \qquad d^2(i,j)=\begin{cases}
    0,&\text{if $i=j$ or $i$ and $j$ are adjacent}\\
    1,&\text{otherwise}  
    \end{cases}    \f$ 
 </td></tr>

 <tr>  
 <td>  
 \f$  \text{Translated adjacency distance}\f$ 
 </td><td>              
 \f$ \qquad d^2(i,j)=\begin{cases}
    0,&\text{if $i=j$}\\
    1-\frac{2}{n},&\text{if $i$ and $j$ are adjacent}\\
    1,&\text{otherwise}
    \end{cases}    \f$ 
 </td></tr>

 <tr> 
<td>
 \f$ \text{Bisection distance} \f$
 </td><td>
  \f$ \qquad d^2(i,j)=\begin{cases}
    0,&\text{if $i=j$}\\
    1-\frac{2}{d(i)+d(j)+2},&\text{if $i$ and $j$ are adjacent}\\
    1,&\text{otherwise}
    \end{cases}
      \f$  
</td></tr>

<tr>
<td> 
 \f$ \mathbb{R}^2\text{distance} \f$
 </td><td>
  \f$ \qquad d^2(i,j)=x^2(i)+y^2(i)
   \f$
</td></tr>

<tr>
<td>
   \f$ \text{Laplacian distance}\f$
 </td><td>
   \f$ \qquad d^2(i,j)=\begin{cases}
    0,&\text{if $i=j$}\\
    2n-d(i)-d(j),&\text{if $i$ and $j$ are adjacent}\\
    2n-d(i)-d(j)+2,&\text{otherwise}
    \end{cases}    \f$ 
</td></tr>

<tr>    
<td>
 \f$ \text{Q distance} \f$
</td><td>
  \f$ \qquad d^2(i,j)=\begin{cases}
    0,&\text{if $i=j$}\\
    1,&\text{if $i$ and $j$ are non adjacent}\\
    1-\frac{1}{\sqrt{d(i)d(j)}},&\text{otherwise}
    \end{cases}    \f$
</td></tr>
 </table> 
*/
